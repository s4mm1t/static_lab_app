# Codex handoff for static_lab

Привет! Это короткая записка для друга, который откроет проект через Codex.

Актуальная ветка: `main`.
Последний проверенный коммит перед этой запиской: `beeaf96`.
Production: https://static-lab-app.vercel.app/

Что важно знать сразу:

- Приложение сейчас называется `static_lab`.
- Мобильный design shell принудительно грузится как `/design/index.html?v=fullbind-20260620-2`.
- `/design/*` отдается с `Cache-Control: no-store`, чтобы Vercel/Safari не держали старую версию интерфейса.
- Нижний nav приложения уже прибит к низу viewport: `position: fixed`, `bottom: 0`, `gapToViewportBottom: 0`.
- Если на iPhone снизу видна белая строка с адресом сайта, это Safari toolbar, а не элемент приложения. Обычный сайт не может физически опустить или скрыть эту браузерную панель; для полного скрытия нужен PWA-режим через "Add to Home Screen".

Перед правками:

- Сначала прочитай `AGENTS.md`: в проекте Next.js 16, и правила/доки могут отличаться от привычных.
- Не меняй nav наугад, пока не отделишь наш app nav от нижней панели Safari.
- Для проверки UI используй iPhone/mobile viewport и проверяй production не только визуально, но и по HTML/JS: root должен ссылаться на `fullbind-20260620-2`, а `public/design/app/main-navdock.jsx` должен содержать solid `bottom: 0` nav.

Команды, которые уже проходили на последнем фиксе:

```bash
npm run lint
npm run build
```

Сообщение от владельца проекта: нужна максимально аккуратная работа, особенно вокруг мобильных верхних/нижних плашек, авторизации и AI coach. Если что-то выглядит как "та же плашка", сначала проверь, это элемент сайта или браузерный UI Safari.
